## Baseline and method

Atomic's v0.80.10 baseline is semantic rather than tree-identical. PR #1881 completed the audited v0.80.10 dependency/source sync at `b63112c4a18dc0361eb0a19f687b9419e8a0a006`; current manifests and the former `pi-0.80.10-artifacts.test.ts` then pinned and verified all 12 workspace declarations plus exact lock artifacts. The upstream range contains 78 commits, 37 of which touch `packages/coding-agent`; its final coding-agent tree changes 93 files (+3,904/-479). Commands used:

```bash
git fetch --tags origin v0.80.10 v0.81.0 v0.81.1
git log --reverse --format='%H%x09%s' v0.80.10..v0.81.1
git log --reverse --format='%H%x09%s' v0.80.10..v0.81.1 -- packages/coding-agent
git diff --name-status v0.80.10..v0.81.1 -- packages/coding-agent
git diff --stat v0.80.10..v0.81.1 -- packages/coding-agent
```

Atomic retains its package/binary/config identity, compatibility aliases, bundled workflows/subagents/MCP/web/intercom packages, DBOS durability, Verbatim Compaction, hashline editing, split interactive engine, provider repairs, and versionless release flow. Cursor support described by this historical audit was removed in issue #1994. Changes below were therefore ported semantically rather than wholesale cherry-picked.

## Deferred additive features

The guardrail allows large features to be skipped when they would destabilize the bug-fix port. Local llama.cpp management is a new six-module built-in extension plus router/process/download UI and conflicts with Atomic's first-party package discovery/isolated engine; expanded usage accounting changes durable session-entry shapes and overlaps Verbatim Compaction plus DBOS extensions; native provider registration adds a second extension contract beside Atomic's compatibility provider API. All three are cleanly absent rather than partially exposed and should be evaluated in dedicated follow-ups. Release archives and upstream SQLite publication are not follow-ups for this port because Atomic intentionally uses different release/durability systems.
