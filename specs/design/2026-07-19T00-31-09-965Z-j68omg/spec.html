Design system · DESIGN.md, applied to this artifact

System reference

1 · Overview — "The Quiet Operator", with one gesture and one sentence

Atomic is a terminal-native control plane for AI coding agents. The interface disappears into the work: glanceable at 2am, dense without clutter, surfacing a saturated color only when something needs attention. The register is Restrained — tinted-neutral surfaces, one accent (Catppuccin Blue) reserved for primary action and selection, status colors mapped one-to-one to orchestrator state, Unicode iconography only, no emoji. Depth is tonal strata only; no shadow, blur, or glow exists in a terminal. Revision 3 holds the startup gesture singular: one 640ms two-half assembly, one manifesto sentence, then silence — everything after the landing screen is status. Working-time motion is illumination, not rotation: activity reads as light moving through a fixed mark via role change, weight, and glyph density, never as a spinning wheel and never as CSS effects Ink cannot render.

  • One-Accent Rule. Blue on ≤10% of the surface at any time; one accented element per simulated frame. In a mark frame, the illumination sweep is the accent — the phase label beside it stays muted, never accented.
  • Status-Is-Truth Rule. Yellow = running, green = complete, red = error, sky = awaiting input, overlay1 = pending. Never decorative. Marks never borrow status hues.
  • Illumination roles. A sweeping cell is accent + bold + denser glyph; the trailing cell is muted; resting cells are overlay1-dim. Under NO_COLOR the density swap alone (│→┃, ░→▒, ♕→♛, 0↔1) carries the sweep — meaning never rides on hue.
  • NO_COLOR compensation. Under data-color="off", hue collapses to text and dims lift to nc-dim/nc-muted so contrast stays ≥4.5:1.
  • Contrast repair tier. All revision-3 captions/kickers use --muted (≈7.4:1 on base) via the .kk tier, because --overlay1 on base measures ≈4.4:1 — below AA for small text. The locked startup keeps its original styling by the byte-preservation mandate.

3 · Typography — weight and color only

The terminal controls font and size; Atomic picks weight and color. Hierarchy is bold/regular × text/textMuted/dim — six combinations cover the entire system (the Weight-and-Tone Rule). Inventing a seventh (italics, underline, blink) is a smell, with one shipped exception: the hidden-thinking label is italic-muted in the product and the artifact mirrors it.

4 · Elevation — stacked tonal strata

No shadows, no blur, no glow — the No-Glow Rule is absolute inside terminal frames, and it is why every "light" effect in the mark study is built from color roles, weight, and glyph density instead of CSS filters. Depth is conveyed exclusively by which solid stratum a component paints. Focus states shift one stratum and recolor the border — that is the entire focus vocabulary.

6 · Do's and Don'ts

Do

  • Use Mocha tokens by role — never inline a hex in a component.
  • Move banner halves in whole-column steps on the fixed grid; every frame must be renderable by Ink as plain cells.
  • Ease out with no overshoot — the halves meet decisively, they do not bounce.
  • Settle everything (assembly, manifesto, spinners, all marks) to a truthful static state within one tick when motion is off or prefers-reduced-motion is set — static, never blank.
  • Carry mark activity through role change + bold + glyph density so NO_COLOR keeps the sweep readable as shape change.
  • Render the manifesto once per session, on the landing screen, in the user's exact punctuation.
  • Keep one shared clock, wall-clock-indexed frames, and precomputed frame sets; pause when the document is hidden.
  • State multi-cell footprint honestly: rows vs the incumbent's 2, drawn as composer displacement, not asserted.
  • Keep selection legible without color — the ▸ marker plus border, roving tabindex, arrow keys.

Don't

  • Don't tween the banner sub-cell — no fractional transform or pixel interpolation; travel is column-quantized or it is not this design.
  • Don't use filter, text-shadow, box-shadow, or gradients inside a terminal frame — Ink cannot render them and the No-Glow Rule forbids them.
  • Don't let a working mark read as rotation — that duplicates the incumbent spinner and fails the concept.
  • Don't present a 5×7 mark as low-cost — every row past two reflows scrollback above the composer on every render.
  • Don't repeat the manifesto as status copy, footer copy, or receipts — one landing, then silence.
  • Don't put voice or marks on anything yellow or red: errors, retries, cancellation, compaction, blockers, approvals stay plain.
  • Don't reintroduce random rotation — whimsical-messages.ts is the anti-pattern being deleted.
  • Don't render concept A without crown + sword + both fleurs; don't trace the Divergent mark for B; don't auto-recommend or auto-condemn F outside the matrix evidence.
  • Don't use square borders or ASCII +---+ art — rounded ╭╮╰╯ or nothing; no emoji.

Usage example

// startup-assembly.ts — proposed shape (design artifact; not production code)
// Frames are plain cell grids; Ink renders exactly these strings.
const ASSEMBLE_MS = 640, FRAME_MS = 80, GAP_MAX = 10, SEAM = 18; // cols 0–17 | 18–35

function gap(t: number): number {          // whole columns, eased out, no bounce
  return Math.max(1, Math.round(GAP_MAX * Math.pow(1 - t, 1.7)));
}

function assemblyFrame(t: number): string[] {  // t in [0,1); t>=1 → settled grid + ░ shadow + meta
  return BANNER_LINES.map(row => {
    const cells = Array(36).fill(" ");
    [...row].forEach((ch, c) => {
      if (ch === " ") return;
      const nc = c < SEAM ? c - gap(t) : c + gap(t);  // halves travel inward
      if (nc >= 0 && nc < 36) cells[nc] = ch;          // clip at the frame edge
    });
    return cells.join("");
  });
}

Known limitations & assumptions

  • Design artifact only. Neither file implements production behavior; the usage example is a proposed shape, not shipped code. Zero runtime dependencies, zero network requests.
  • Assumption — timing envelope reading. The 500–800ms envelope is treated as governing the mark entrance only (assembly = 640ms). The manifesto beat extends the full startup sequence to ~1040ms. If the envelope is meant to bound the entire sequence, compress the beat offsets (e.g. +160/+280ms) — the choreography is unaffected.
  • Assumption — punctuation verdict. Rendering A (the user's exact comma splice) is adopted; rendering B (em dash) is the one tested alternative and is not shipped. Reversing that is a one-string change.
  • Assumption — default gallery selection. The sheet loads with A1 selected (first tile, neutral). The recommendation itself points at B1; pre-selecting the verdict is a one-line change if preferred.
  • Working-mark adoption is explicitly not decided. The study's only binding outputs are: keep Braille in the one-line slot, E disqualified, F not recommended, B1 named as the candidate if a multi-line surface ever wants a mark. No production component work is authorized by this spec.
  • Matrix scores are designer judgments backed by the live frames and the stated evidence (footprint math, NO_COLOR behavior, reference legibility) — not user-tested data. The cultural/legal column records exposure (heraldic association, faction-mark adjacency, hacker-skull cliché), not legal advice.
  • Contrast figures are computed (WCAG 2.x formula from token hexes), not screen-measured. Carried-forward rev-2 content keeps its original --overlay1 captions (≈4.4:1) by the byte-preservation mandate; only revision-3 content uses the repaired .kk tier.
  • Line anchors drift. File:line anchors (working-status.ts:29–41, interactive-startup.ts:147, etc.) were verified at review time; re-verify before editing. Note the corrected path: components/chat-session-host-utils.ts (the rev-2 spec omitted the components/ segment).
  • Iframe keyboard focus. The embedded preview's key deck and gallery need focus inside the frame; reviewers evaluating keyboard behavior should open the file URL directly.
  • Reference authority. The five user-supplied images are the composition authority for concepts A and B and outrank DESIGN.md on those compositions only; all rendering still passes through Mocha roles and Unicode cells. The images are not bundled into this artifact.
