Microcopy system · family, mapping, law

  
Six lanes, one pure function, a hard never-list

  
Each lane traces to a line of README.md positioning. The mapping is a total, pure function of the latest runtime event — publishable, testable, and boring to maintain. That is the point.

Mapping rules

  
    
    
  1. Precedence. Compaction and statusMessage outrank any lane label (chat-session-host-rendering.ts:83–85). A lane label never renders while compacting.
    2. 
    
  2. Purity. label = phaseLabel(latestEvent, toolClass). Same event, same label, every run. No Math.random(), no timers, no rotation while a phase persists.
    3. 
    
  3. Totality. Any unmapped event falls back to On it. The function can never throw or go blank.
    4. 
    
  4. Length. ≤24 characters, ASCII plus narrow Unicode only. Spinner + gutter + longest label = 27 columns — fits the 64-col working line with room to spare.
    5. 
    
  5. Punctuation. No ellipsis on the spinner line — the spinner is the progress mark. The hidden-thinking slot keeps its shipped ellipsis; it has no spinner.
    6. 
    
  6. Deletability. Every label must be removable without information loss. Facts live in status lines and receipts; the label carries voice only.
    7. 
  

  
What never gets personality

  
    
    
  • Not on errors. ✗ lines state the command and the exit code, nothing else.
    • 
    
  • Not on destructive actions. Approvals name the risk in plain words and wait.
    • 
    
  • Not on blockers. ⊘ blocked by unit tests names the dependency, full stop.
    • 
    
  • Not on cancellation. Compaction cancelled. A fact, not commentary.
    • 
    
  • Not on retries. Retrying (2/5) in 8s... (esc Cancel) — count, countdown, control.
    • 
    
  • Not on compaction. Compacting context... outranks every label — by code, not by policy.
